21 1.4.0 (Jan 2010) Reduced the default value for the q floating point image
22  quantization parameter from 16 to 4. This results in about 50% better
23  compression (from about 4.6x to 6.4) with no lost of significant information
24  (with the new subtractive dithering enhancement). Replaced the code for
25  generating temporary filenames to make the code more portable (to Windows).
26  Replaced calls to the unix 'access' and 'stat' functions with more portable
27  code. When unpacking a file, write it first to a temporary file, then
28  rename it when finished, so that other tasks cannot try to read the file
29  before it is complete.
30 1.3.0 (Oct 2009) added randomization to the dithering pattern so that
31  the same pattern is not used for every image; also added an option
32  for losslessly compressing floating point images with GZIP for test
33  purposes (not recommended for general use). Also added support for
34  reading the input FITS file from the stdin file streams.
35 1.2.0 (Sept 2009) added subtractive dithering feature (in CFITSIO) when
36  quantizing floating point images; When packing an IRAF .imh + .pix image,
37  the file name is changed to FILE.fits.fz, and if the original file is
